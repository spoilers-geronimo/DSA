Formal Definition

A String is a finite, ordered sequence of characters drawn from a defined alphabet (commonly Unicode). In most programming languages, a string is internally represented as a contiguous array of character codes. Three critical properties govern how strings behave: immutability (in many languages, a string value cannot be modified after creation), slicing (extracting a contiguous sub-sequence by index range), and hashing (mapping the entire character sequence to a fixed-size integer for fast equality checks and dictionary lookups).
2

Formal Explanation & Operations

🔒 Immutability

In JavaScript, Python, Java, and Go, strings are immutable: once a string is allocated in memory, its characters cannot be changed. Any "modification" creates a brand-new string object. This enables safe sharing, caching, and hashing.

✂️ Slicing

A slice extracts a substring from index start (inclusive) to end (exclusive). In most languages this copies the characters into a new string in O(k) time, where k is the length of the slice.

#️⃣ Hashing

A hash function scans every character and produces a fixed-size integer fingerprint. Equal strings always produce the same hash. This enables O(1) average-case lookup in hash maps and sets.

Operation Description Time Notes
s[i] Access character at index i O(1) Direct index, like an array
s.length Number of characters O(1) Stored as a field, not computed
Slice s[i:j] Copy characters from index i to j−1 O(k) k = length of slice; copies memory
Concatenation a + b Combine two strings into a new one O(n+m) Allocates new memory of size n+m
Search indexOf Find first occurrence of a substring O(n·m) n = string len, m = pattern len (naïve)
Hash hash(s) Compute integer fingerprint O(n) Usually computed once & cached
Equality s1 === s2 Compare two strings character by character O(n) O(1) if hashes differ and are pre-computed
⚠️

The concatenation trap: Joining n strings in a loop with result += s[i] is O(n²) because each + allocates a new string. Use Array.join() or a StringBuilder (O(n) total) instead.

Code Snippet

Three focused examples — one per core concept: immutability, slicing, and hashing.

JavaScript — Immutability 
// ─── Strings are immutable in JavaScript ────────────────────────
const original = "hello";

// You CANNOT modify a character in place:
// original[0] = 'H';  ← silently fails (or throws in strict mode)
console.log(original[0]);   // "h"  (still lowercase)

// "Modifying" a string always creates a NEW string in memory:
const modified = original.toUpperCase();
console.log(original);       // "hello"   ← unchanged
console.log(modified);       // "HELLO"   ← brand new string object

// Each operation returns a new string; the original is untouched:
const s = "  trim me  ";
console.log(s.trim());       // "trim me"  ← new string
console.log(s);              // "  trim me  "  ← original unchanged

// ⚠️  The O(n²) concatenation trap — AVOID this:
let bad = "";
const words = ["one", "two", "three", "four"];
for (const w of words) bad += w + " ";  // O(n²) — new string every iteration

// ✅  The correct O(n) approach — use join():
const good = words.join(" ");           // "one two three four"
JavaScript — Slicing 
// ─── String Slicing ─────────────────────────────────────────────
const s = "Hello, World!";
//         0123456789...

// slice(start, end): characters from index start up to (not including) end
console.log(s.slice(0, 5));    // "Hello"    — indices 0,1,2,3,4
console.log(s.slice(7, 12));   // "World"    — indices 7–11
console.log(s.slice(-6));      // "orld!"    — negative: count from end
console.log(s.slice(7));       // "World!"   — no end = slice to end

// Slices are copies — mutating one does NOT affect the other
const sub = s.slice(0, 5);
console.log(sub);              // "Hello"
console.log(s);                // "Hello, World!" — original untouched

// Practical: reverse a string using slice + split + join
function reverseString(str) {
  return str.split("").reverse().join("");
}
console.log(reverseString("racecar"));  // "racecar" (palindrome)
console.log(reverseString("hello"));    // "olleh"

// Practical: check if a string is a palindrome — O(n)
function isPalindrome(str) {
  const clean = str.toLowerCase().replace(/[^a-z0-9]/g, "");
  let left = 0, right = clean.length - 1;
  while (left < right) {
    if (clean[left] !== clean[right]) return false;
    left++; right--;
  }
  return true;
}
console.log(isPalindrome("A man a plan a canal Panama")); // true
console.log(isPalindrome("hello"));                      // false
JavaScript — Hashing & Frequency Maps 
// ─── Hashing: djb2 hash function (manual implementation) ────────
// Shows the principle: scan every char, mix its code into a number.
function hashString(str) {
  let hash = 5381;                        // Magic seed
  for (let i = 0; i < str.length; i++) {
    hash = (hash * 33) ^ str.charCodeAt(i);
    hash = hash & hash;                   // Keep it 32-bit
  }
  return Math.abs(hash);
}

console.log(hashString("hello"));       // e.g. 210700827
console.log(hashString("hello"));       // same — deterministic
console.log(hashString("Hello"));       // different — case-sensitive

// ─── Practical: Anagram check using a frequency map ─────────────
// Two strings are anagrams if they contain the same characters
// with the same frequencies. Solved in O(n) using a hash map.
function isAnagram(s, t) {
  if (s.length !== t.length) return false;

  const freq = {};

  for (const ch of s) {
    freq[ch] = (freq[ch] ?? 0) + 1;   // Count characters in s
  }
  for (const ch of t) {
    if (!freq[ch]) return false;      // Missing character
    freq[ch]--;
  }
  return true;
}

console.log(isAnagram("listen", "silent")); // true
console.log(isAnagram("hello",  "world"));  // false
console.log(isAnagram("rat",    "car"));    // true

Plain English Analogy

🪨

Immutability — A Message Engraved in Stone

Imagine you carve a message into a stone tablet: "hello". You can read any letter — but you can't erase and recarve just one character without destroying the whole tablet. If you want to change it to "Hello", you have to carve a brand-new tablet. The old one still exists, unchanged.

That's immutability. The computer keeps the original string in memory forever until no one needs it anymore. This makes strings safe to share between different parts of a program — nobody can secretly modify the value under your nose.

✂️

Slicing — Photocopying Part of a Book

If you photocopy pages 7 to 12 of a book, you get a new document that contains only those pages. The original book still exists, fully intact. Your photocopy is independent — writing on it doesn't affect the original.

String slicing works the same way. s.slice(7, 12) photocopies characters 7 through 11 into a brand-new string. The original s is untouched.

#️⃣

Hashing — A Locker Number from Your Name

Imagine a gym that assigns you a locker number by running your name through a special formula. Every time you type in "Alice", you always get locker 42. This lets the gym find your locker instantly — without searching every locker one by one.

String hashing does exactly this: it turns a long string of characters into a small fixed number so the computer can find it in a big data structure (like a dictionary) in O(1) time, instead of checking every entry.

  • "Alice" → locker 42 = hash("alice") → 42
  • Two people with the same locker = a hash collision
  • Changing one letter changes the locker number = hash sensitivity

Real-World Coding Applications

String operations are at the core of virtually every software system humans interact with daily.

🔑

Password Hashing

When you create a password, it's hashed (with a salt) before storage. The server never stores the actual password — only its hash. On login, the hash is compared, never the plain text.

🔍

Search Engines

Google tokenises, slices, and hashes billions of web pages. Substring search and string matching algorithms (like KMP and Rabin-Karp) power search relevance and autocomplete.

🌐

URL Routing

Web frameworks split and slice URL paths like /users/42/profile into segments to route requests to the correct handler function.

🧬

DNA Sequence Analysis

Genomic data is literally stored and processed as strings over a 4-character alphabet (A, T, C, G). Substring matching finds genes and mutations in sequences billions of characters long.

🪪

Deduplication & Caching

In-memory caches (like Redis) use string hashing to detect duplicate requests or cache keys in O(1). Python interns short strings so identical literals share one memory address.

📝

Text Editors & Diff Tools

Git's diff algorithm breaks files into lines (string slices) and computes the longest common subsequence between them to show exactly what changed.
