Formal Definition

A Stack is an abstract data type that represents a collection of elements governed by the Last-In, First-Out (LIFO) principle: the most recently inserted element is always the first to be removed. A stack exposes three primary operations — push (insert onto the top), pop (remove from the top), and peek (read the top element without removing it) — all of which execute in O(1) time. Access to arbitrary elements is not permitted; elements can only be interacted with at the top of the stack.

A stack can be implemented using either an array (with a top-pointer tracking the last element) or a linked list (where push/pop operate on the head). Both give the same O(1) guarantees; the choice affects memory behaviour, not asymptotic complexity.

2

Plain English Explanation & Operations

A stack enforces a strict rule: you can only ever touch the top. There's no jumping to the middle, no peeking at the bottom. This constraint makes a stack simpler and faster than most other structures — and it turns out that a huge number of real computing problems are naturally "top-first".

OperationDescriptionTimeSpace
push(x) Add element x onto the top of the stack O(1) O(1)
pop() Remove and return the top element O(1) O(1)
peek() Return the top element without removing it O(1) O(1)
isEmpty() Return true if the stack has no elements O(1) O(1)
size() Return the number of elements O(1) O(1)
Access by index Not supported — stacks do not allow random access N/A N/A
💡

Why O(1) for everything? A stack only ever operates on a single, known position — the top. There's no searching, no shifting, no resizing in normal operation. It's the computational equivalent of a notepad where you only ever read or write the top sticky note.

⚠️

Stack Overflow: When a recursive function calls itself too many times, each call pushes a new frame onto the program's call stack. If the recursion has no proper base case, the stack grows until memory is exhausted — causing the famous "stack overflow" error.

Code Snippet

The first snippet shows a stack built on top of a JavaScript array (the idiomatic approach). The second builds one from scratch using a linked list to show how the internals work.

JavaScript — Stack using Array 
// ─── Stack backed by a dynamic array ────────────────────────────
class Stack {
  constructor() {
    this.items = [];
  }

  // O(1) amortised — add to end of array
  push(value) {
    this.items.push(value);
  }

  // O(1) — remove from end of array
  pop() {
    if (this.isEmpty()) throw new Error("Stack underflow");
    return this.items.pop();
  }

  // O(1) — read top without removing
  peek() {
    if (this.isEmpty()) throw new Error("Stack is empty");
    return this.items[this.items.length - 1];
  }

  isEmpty() { return this.items.length === 0; }
  size()    { return this.items.length; }
}

// ─── Usage ───────────────────────────────────────────────────────
const s = new Stack();
s.push(10);
s.push(20);
s.push(30);
console.log(s.peek());   // 30  — top of stack
console.log(s.pop());    // 30  — removed
console.log(s.peek());   // 20  — new top
console.log(s.size());   // 2
JavaScript — Stack using Linked List 
// ─── Stack backed by a singly linked list ───────────────────────
// Push and pop operate on the HEAD — always O(1), no resizing.
class StackNode {
  constructor(value) {
    this.value = value;
    this.next  = null;
  }
}

class LinkedStack {
  constructor() {
    this.top    = null;
    this._size  = 0;
  }

  push(value) {
    const node  = new StackNode(value);
    node.next   = this.top;   // new node points to old top
    this.top    = node;        // advance top pointer
    this._size++;
  }

  pop() {
    if (!this.top) throw new Error("Stack underflow");
    const val   = this.top.value;
    this.top    = this.top.next; // advance past old top
    this._size--;
    return val;
  }

  peek()    { return this.top?.value; }
  isEmpty() { return this.top === null; }
  size()    { return this._size; }
}

Plain English Analogy

🍽️

A Stack of Plates in a Cafeteria

Imagine a cafeteria with a spring-loaded plate dispenser. You can only ever take the top plate from the pile, and you can only ever place a new plate on top. You can't reach in and grab the plate from the middle — the whole structure would collapse.

That's exactly how a stack works. The last plate put on (pushed) is the first one taken off (popped). This is LIFO — Last In, First Out.

When your phone app has an "undo" button, each action you take is a plate being placed on the pile. Pressing undo lifts the top plate — the most recent action — and reverses it. You can't undo the action you did three steps ago without first undoing the two actions on top of it.

  • Plate = an element in the stack
  • Place a plate on top = push
  • Take the top plate = pop
  • Glance at the top plate without touching it = peek
  • Empty dispenser = stack underflow (popping an empty stack)

Real-World Coding Applications

Stacks are everywhere in software — often hidden inside the runtime itself.

📞

The Call Stack

Every programming language runtime uses a call stack to track active function calls. Each invocation pushes a frame; return pops it. "Stack overflow" literally means this stack ran out of memory.

↩️

Undo / Redo

Text editors maintain two stacks — an undo stack and a redo stack. Each edit is pushed onto the undo stack. Pressing Ctrl+Z pops from undo and pushes to redo.

🔣

Expression Parsing

Compilers use stacks to evaluate mathematical expressions and convert between infix, prefix, and postfix notation. Your calculator app does this with every calculation.

🌐

Browser Back Button

Your browser's back navigation is a stack. Every page you visit is pushed. Clicking Back pops the current page and returns to the previous one.

🏗️

Depth-First Search (DFS)

Graph and tree DFS algorithms use a stack (either explicitly or via recursion's call stack) to track which nodes to visit next — diving deep before backtracking.

🔒

Syntax Validation

Linters, IDEs, and parsers use stacks to check balanced brackets, HTML tags, and XML. Every ( is pushed; every ) is popped and matched.

Classic Interview Question

🔣 Valid Parentheses

Given a string containing only the characters (, ), {, }, [, and ], determine if the input string is valid. A string is valid if every opening bracket is closed by the same type of bracket in the correct order.

Examples: "()[]{}" → true  |  "([{}])" → true  |  "(]" → false  |  "([)]" → false

⚡ Easy — LeetCode #20, asked at every major company

Why this question? It's the definitive stack problem. The solution is short, elegant, and only works if you truly understand that a stack naturally models the "most recent opening bracket" — the ideal candidate for LIFO access.

The approach:

  • 1
    Create an empty stack and a mapping of closing → opening brackets.
  • 2
    Loop through each character in the string.
  • 3
    If it's an opening bracket ((, [, {), push it onto the stack.
  • 4
    If it's a closing bracket, check if the stack's top is the matching opener. If not, return false.
  • 5
    After the loop, the stack must be empty — any leftover openers mean unmatched brackets.
JavaScript — Valid Parentheses 
// ─── Valid Parentheses — O(n) time, O(n) space ──────────────────
function isValid(s) {
  const stack = [];
  const map = {
    ')': '(',
    ']': '[',
    '}': '{',
  };

  for (const ch of s) {
    if (ch === '(' || ch === '[' || ch === '{') {
      stack.push(ch);             // opening — push onto stack
    } else {
      if (stack.pop() !== map[ch]) {
        return false;            // mismatch or empty stack
      }
    }
  }

  return stack.length === 0;   // true only if all openers were matched
}

// ─── Test cases ──────────────────────────────────────────────────
console.log(isValid("()[]{}"));  // true
console.log(isValid("([{}])"));  // true
console.log(isValid("(]"));      // false — wrong closing bracket
console.log(isValid("([)]"));    // false — wrong order
console.log(isValid("{"));        // false — unmatched opener

// ─── Walkthrough: "([{}])" ───────────────────────────────────────
// '('  → push → stack: ['(']
// '['  → push → stack: ['(', '[']
// '{'  → push → stack: ['(', '[', '{']
// '}'  → pop '{' — matches map['}'] ✓ → stack: ['(', '[']
// ']'  → pop '[' — matches map[']'] ✓ → stack: ['(']
// ')'  → pop '(' — matches map[')'] ✓ → stack: []
// stack.length === 0 → return true ✓
✅

Interview tip: When you call stack.pop() on an empty array in JavaScript, it returns undefined — which won't match any bracket, so the check handles the empty-stack edge case automatically. Point this out: interviewers love candidates who think about edge cases without prompting. Also mention the O(n) space for the stack in your complexity analysis.
