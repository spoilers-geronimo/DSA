Heaps & Priority Queues

Formal Definition

A Heap is a specialised complete binary tree that satisfies the heap property: in a min-heap, every node's value is less than or equal to the values of its children, making the root the global minimum; in a max-heap, every node's value is greater than or equal to its children, making the root the global maximum. Heaps are typically stored as a flat array using the index formulas left child = 2i + 1, right child = 2i + 2, and parent = ⌊(i−1)/2⌋. A Priority Queue is the abstract data type that a heap implements — a collection where the element with the highest (or lowest) priority is always dequeued first.

Heaps do not sort all elements — they only guarantee the root is the minimum (or maximum). This partial ordering is exactly what's needed for efficient priority-based access, and it's what makes heap operations O(log n) rather than O(n log n).

2

Plain English Explanation & Operations

Think of a heap as a self-organising structure with one guarantee: the most important element is always at the top. Every insertion or removal triggers a "bubbling" process that restores this guarantee in O(log n) steps — proportional to the tree's height.

⬇️ Min-Heap

  • Root is always the smallest value
  • Every parent ≤ both its children
  • peek() returns the minimum in O(1)
  • Used for: Dijkstra's, scheduling, merge K lists
  • Python's heapq is a min-heap by default

⬆️ Max-Heap

  • Root is always the largest value
  • Every parent ≥ both its children
  • peek() returns the maximum in O(1)
  • Used for: heap sort, Top-K largest problems
  • Negate values to simulate max-heap in Python

Two core internal operations keep the heap valid after every change:

Bubble Up (sift up): After inserting a new element at the end of the array, compare it with its parent. If it violates the heap property, swap them. Repeat until the property is restored or the root is reached. This climbs up the tree — O(log n).

Bubble Down (sift down): After removing the root (replacing it with the last element), compare the new root with its children. Swap with the smaller child (min-heap) or larger child (max-heap) if needed. Repeat downward — O(log n).

OperationDescriptionTimeSpace
peek() Return the root (min or max) without removing O(1) O(1)
insert(x) Add element, bubble up to restore heap property O(log n) O(1)
extractMin/Max() Remove root, bubble down last element O(log n) O(1)
heapify(array) Build a heap from an unsorted array in-place O(n) O(1)
Search Find an arbitrary element (heap is not sorted) O(n) O(1)
Heap Sort Build heap, then repeatedly extract root O(n log n) O(1)
💡

Why is heapify O(n) and not O(n log n)? Building a heap bottom-up means most nodes only sift down a short distance — nodes near leaves do almost no work. The mathematical sum of all sift-down work across all nodes converges to O(n), not O(n log n). This is a crucial insight for heap sort analysis.

⚠️

Heaps are not sorted. A min-heap guarantees the root is the minimum, but the rest of the elements are only partially ordered. You cannot index into a heap to find the k-th smallest element in O(1) — that requires extracting the minimum k times at O(k log n) total.

Code Snippet

The first snippet builds a min-heap from scratch as an array, fully implementing insert and extract with bubble-up and bubble-down. The second shows a practical priority queue pattern used directly in interview problems.

JavaScript — Min-Heap from Scratch 
class MinHeap {
  constructor() {
    this.heap = [];
  }

  // Index helpers
  _parent(i) { return Math.floor((i - 1) / 2); }
  _left(i)   { return 2 * i + 1; }
  _right(i)  { return 2 * i + 2; }

  _swap(i, j) {
    [this.heap[i], this.heap[j]] = [this.heap[j], this.heap[i]];
  }

  // O(1) — peek at minimum without removing
  peek() { return this.heap[0]; }

  // O(log n) — insert and bubble up
  insert(val) {
    this.heap.push(val);
    this._bubbleUp(this.heap.length - 1);
  }

  _bubbleUp(i) {
    while (i > 0) {
      const p = this._parent(i);
      if (this.heap[p] <= this.heap[i]) break; // heap property satisfied
      this._swap(p, i);
      i = p;
    }
  }

  // O(log n) — remove and return the minimum
  extractMin() {
    if (this.heap.length === 0) return null;
    if (this.heap.length === 1) return this.heap.pop();

    const min = this.heap[0];
    this.heap[0] = this.heap.pop(); // move last element to root
    this._bubbleDown(0);
    return min;
  }

  _bubbleDown(i) {
    const n = this.heap.length;
    while (true) {
      let smallest = i;
      const l = this._left(i);
      const r = this._right(i);

      if (l < n && this.heap[l] < this.heap[smallest]) smallest = l;
      if (r < n && this.heap[r] < this.heap[smallest]) smallest = r;

      if (smallest === i) break;     // already the smallest — done
      this._swap(i, smallest);
      i = smallest;
    }
  }

  size() { return this.heap.length; }
}

// ─── Usage ───────────────────────────────────────────────────────
const h = new MinHeap();
[5, 3, 8, 1, 4].forEach(v => h.insert(v));
console.log(h.peek());         // 1   — min always at root
console.log(h.extractMin());   // 1
console.log(h.extractMin());   // 3
console.log(h.extractMin());   // 4
JavaScript — Heap Sort & Top-K Pattern 
// ─── Find Kth Largest using a Min-Heap ───────────────────────────
// Maintain a min-heap of size K. If a new element is larger than
// the heap's min (root), evict the root and insert the new element.
// After processing all elements, the root is the Kth largest.
function kthLargest(nums, k) {
  const heap = new MinHeap();

  for (const num of nums) {
    heap.insert(num);
    if (heap.size() > k) {
      heap.extractMin();   // discard smallest — keep only top K
    }
  }

  return heap.peek();   // root of size-K heap = Kth largest
}

console.log(kthLargest([3, 2, 1, 5, 6, 4], 2));  // 5
console.log(kthLargest([3, 2, 3, 1, 2, 4, 5, 5, 6], 4)); // 4

// ─── Heap Sort (in-place, O(n log n)) ────────────────────────────
function heapSort(arr) {
  const n = arr.length;

  // Phase 1: Build a max-heap in-place — O(n)
  for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
    siftDown(arr, i, n);
  }

  // Phase 2: Extract max to end, shrink heap — O(n log n)
  for (let end = n - 1; end > 0; end--) {
    [arr[0], arr[end]] = [arr[end], arr[0]];  // swap root (max) to end
    siftDown(arr, 0, end);                      // restore heap on remaining
  }
  return arr;
}

function siftDown(arr, i, n) {
  while (true) {
    let largest = i;
    const l = 2 * i + 1, r = 2 * i + 2;
    if (l < n && arr[l] > arr[largest]) largest = l;
    if (r < n && arr[r] > arr[largest]) largest = r;
    if (largest === i) break;
    [arr[i], arr[largest]] = [arr[largest], arr[i]];
    i = largest;
  }
}

console.log(heapSort([4, 10, 3, 5, 1]));  // [1, 3, 4, 5, 10]

Plain English Analogy

🏥

A Hospital Emergency Room Triage System

In an A&E department, patients don't get seen in the order they arrive — they get seen in order of medical urgency. A patient who just had a heart attack is seen before someone with a sprained ankle, even if the ankle patient arrived first. This is a priority queue in action.

The nurse at triage is the heap. Every time a new patient arrives (insert), the nurse assesses their severity and places them in the correct position in the mental priority order — taking O(log n) comparisons. When a doctor is free (extract), the nurse instantly knows who to send next: the most critical patient (peek → O(1)), and reorganises the queue to reflect the next most critical (bubble down → O(log n)).

  • Patient = element in the heap
  • Medical urgency score = the priority / value
  • The most critical patient = the root (min or max)
  • Triage nurse assigning priority = bubble up on insert
  • Calling next patient = extractMin / extractMax
  • Re-evaluating the queue after someone is called = bubble down

Real-World Coding Applications

Heaps appear wherever you need fast access to the extreme element — minimum, maximum, or highest priority.

🧭

Dijkstra's Shortest Path

Dijkstra's algorithm uses a min-heap (priority queue) to always expand the closest unvisited node next. Without a heap, this is O(V²); with a heap, it's O((V + E) log V).

💻

OS Process Scheduling

Operating systems use priority queues to schedule CPU-bound processes. Higher-priority processes are scheduled first; a heap ensures the scheduler picks the next process in O(log n).

🔗

Merge K Sorted Lists

A min-heap of size K simultaneously tracks the current minimum across K sorted lists. At each step, extract the min and push the next element from that list. Total: O(n log K).

📊

Top-K Problems

Finding the K largest elements in a stream of data uses a size-K min-heap. This is a core data engineering pattern used in real-time analytics, leaderboards, and recommendation systems.

🗜️

Huffman Encoding

Huffman compression builds an optimal prefix-free code by repeatedly merging the two lowest-frequency symbols — exactly a min-heap extract-min pattern applied repeatedly.

🤖

A* Pathfinding

The A* algorithm (used in games and robotics) uses a min-heap ordered by g(n) + h(n) (cost + heuristic). It always expands the most promising node first, faster than Dijkstra's on goal-directed searches.

Classic Interview Question

🔢 Find the Kth Largest Element in an Array

Given an integer array nums and an integer k, return the k-th largest element in the array. Note: it is the k-th largest in sorted order, not the k-th distinct element.

Example: nums = [3,2,1,5,6,4], k = 2 → 5 (the 2nd largest)

🔥 Medium — LeetCode #215 · asked at Amazon, Google, Facebook

Why this question? It's the definitive heap interview problem. There are three approaches — O(n log n) sort, O(n log k) min-heap, and O(n) average quickselect — and knowing all three shows deep algorithmic understanding. In practice, the heap solution is the most interview-friendly because it's clear, provably correct, and directly uses heap semantics.

Three approaches:

  • 1
    Sort descending: return nums.sort((a,b) => b-a)[k-1]. O(n log n). Simple but wastes effort sorting everything.
  • 2
    Min-heap of size K: Maintain a heap with at most K elements. If a new number is larger than the root, pop the root and push the new number. After the loop, the root is the K-th largest. O(n log k) — optimal for streaming data.
  • 3
    Quickselect: Partition-based, similar to quicksort's partition step. O(n) average, O(n²) worst case. Most efficient on average but harder to implement correctly under pressure.
JavaScript — Kth Largest Element (Min-Heap) 
// ─── Approach 2: Min-Heap of size K — O(n log k) ────────────────
// Key insight: in a size-K min-heap, the ROOT is always the K-th largest
// seen so far — everything below the root is larger than it.
function findKthLargest(nums, k) {
  const heap = new MinHeap();  // (use the MinHeap class from snippet 1)

  for (const num of nums) {
    heap.insert(num);
    if (heap.size() > k) {
      heap.extractMin();  // evict smallest — keep only top K candidates
    }
  }

  return heap.peek();  // root of size-K heap = K-th largest overall
}

// ─── Walkthrough: nums=[3,2,1,5,6,4], k=2 ────────────────────────
// Insert 3 → heap: [3]
// Insert 2 → heap: [2,3]
// Insert 1 → heap: [1,3,2] size>2 → extract 1 → heap: [2,3]
// Insert 5 → heap: [2,3,5] size>2 → extract 2 → heap: [3,5]
// Insert 6 → heap: [3,5,6] size>2 → extract 3 → heap: [5,6]
// Insert 4 → heap: [4,6,5] size>2 → extract 4 → heap: [5,6]
// peek() → 5  ✓  (5 is the 2nd largest in [1,2,3,4,5,6])
console.log(findKthLargest([3,2,1,5,6,4], 2));  // 5
console.log(findKthLargest([3,2,3,1,2,4,5,5,6], 4)); // 4

// ─── Approach 3: Quickselect — O(n) average ──────────────────────
function findKthLargestQS(nums, k) {
  const target = nums.length - k;  // k-th largest = (n-k)-th smallest

  function partition(lo, hi) {
    const pivot = nums[hi];
    let store = lo;
    for (let i = lo; i < hi; i++) {
      if (nums[i] <= pivot) {
        [nums[store], nums[i]] = [nums[i], nums[store]];
        store++;
      }
    }
    [nums[store], nums[hi]] = [nums[hi], nums[store]];
    return store;
  }

  function select(lo, hi) {
    if (lo >= hi) return nums[lo];
    const pivot = partition(lo, hi);
    if      (pivot === target) return nums[pivot];
    else if (pivot  < target) return select(pivot + 1, hi);
    else                      return select(lo, pivot - 1);
  }

  return select(0, nums.length - 1);
}
✅

Interview tip: Lead with the heap approach — it's the most natural given the context, handles streaming data, and is easiest to reason about. Compare all three complexities: O(n log n) sort, O(n log k) heap (best when k ≪ n), O(n) average quickselect. The heap solution also directly extends to harder variants: "Top K frequent elements", "K closest points to the origin", and "Merge K sorted lists" — all classic problems that use the exact same size-K min-heap pattern.
