Formal Definition

A Queue is a linear abstract data type that enforces the First In, First Out (FIFO) principle: elements are inserted at one end (the rear) and removed from the opposite end (the front). A Circular Queue uses a fixed-size array whose rear and front pointers wrap around using modular arithmetic, eliminating wasted space. A Deque (Double-Ended Queue) generalises a queue by permitting insertion and deletion at both ends in O(1), making it simultaneously a stack and a queue.
2

Formal Explanation & Operations

→ Standard Queue

  • FIFO: first in, first out
  • Enqueue at rear, dequeue from front
  • Implemented via array or linked list
  • Array version wastes space as front moves
  • Linked list version uses O(n) memory, no waste

↻ Circular Queue

  • Fixed capacity, no wasted slots
  • front = (front + 1) % capacity
  • rear = (rear + 1) % capacity
  • Full when (rear+1) % cap = front
  • Ideal for bounded buffers

⇄ Deque

  • Insert/delete at both front and rear
  • Superset of stack and queue
  • Backed by a doubly linked list or array
  • Used in sliding-window algorithms
  • JS Array works as a deque
Operation Standard Circular Deque Notes
enqueue(x) / pushBack(x) O(1) O(1) O(1) Add to rear
dequeue() / popFront() O(1) O(1) O(1) Remove from front
pushFront(x) ✗ ✗ O(1) Deque only
popBack() ✗ ✗ O(1) Deque only
peek() / front() O(1) O(1) O(1) View without removing
isEmpty() O(1) O(1) O(1) Check if size = 0
Access by index O(n) O(n) O(n) Not a primary operation
💡

Queue vs Stack in one sentence: A stack lets the last person in go first (LIFO). A queue lets the first person in go first (FIFO). Same operations, opposite ordering guarantees — choose based on whether fairness or recency matters.

Code Snippet

A linked-list-backed standard queue, a circular queue with modular pointer arithmetic, and a practical deque-based sliding window maximum problem.

JavaScript — Standard Queue (linked-list backed)
// ─── Node ─────────────────────────────────────────────────────────
class QNode {
  constructor(data) { this.data = data; this.next = null; }
}

// ─── Queue — enqueue at rear, dequeue from front, both O(1) ──────
class Queue {
  constructor() {
    this.front = null;  // Next item to be removed
    this.rear  = null;  // Last item added
    this.size  = 0;
  }

  // Enqueue: add to the rear — O(1)
  enqueue(data) {
    const node = new QNode(data);
    if (!this.rear) {
      this.front = this.rear = node;
    } else {
      this.rear.next = node;   // Link new node at rear
      this.rear = node;
    }
    this.size++;
  }

  // Dequeue: remove from the front — O(1)
  dequeue() {
    if (this.isEmpty())
      throw new Error("Queue is empty");
    const val = this.front.data;
    this.front = this.front.next;   // Move front pointer forward
    if (!this.front) this.rear = null;
    this.size--;
    return val;
  }

  peek()    { return this.front?.data; }
  isEmpty() { return this.size === 0; }

  toArray() {
    const result = []; let curr = this.front;
    while (curr) { result.push(curr.data); curr = curr.next; }
    return result;
  }
}

const q = new Queue();
q.enqueue("Alice"); q.enqueue("Bob"); q.enqueue("Carol");
console.log(q.toArray());   // ["Alice", "Bob", "Carol"]
console.log(q.dequeue());   // "Alice" — first in, first out
console.log(q.peek());      // "Bob"   — now at front
JavaScript — Circular Queue (fixed-size array)
// ─── Circular Queue ───────────────────────────────────────────────
// Uses modular arithmetic so the array "wraps around".
// Eliminates the wasted-space problem of a naive array queue.
class CircularQueue {
  constructor(capacity) {
    this.capacity = capacity;
    this.data     = new Array(capacity);
    this.front    = 0;
    this.rear     = 0;  // Points to next empty slot
    this.size     = 0;
  }

  enqueue(val) {
    if (this.isFull())
      throw new Error("Queue is full");
    this.data[this.rear] = val;
    this.rear = (this.rear + 1) % this.capacity; // Wrap around
    this.size++;
  }

  dequeue() {
    if (this.isEmpty())
      throw new Error("Queue is empty");
    const val = this.data[this.front];
    this.front = (this.front + 1) % this.capacity; // Wrap around
    this.size--;
    return val;
  }

  isFull()  { return this.size === this.capacity; }
  isEmpty() { return this.size === 0; }
  peek()    { return this.data[this.front]; }
}

const cq = new CircularQueue(4);
cq.enqueue(1); cq.enqueue(2); cq.enqueue(3);
console.log(cq.dequeue());   // 1 — front slot freed
cq.enqueue(4); cq.enqueue(5); // 5 wraps into slot 0!
console.log(cq.peek());      // 2 — new front
JavaScript — Deque & Sliding Window Maximum
// ─── Deque (double-ended queue) ───────────────────────────────────
class Deque {
  constructor() { this.items = []; }

  pushFront(x) { this.items.unshift(x); }   // Add to front
  pushBack(x)  { this.items.push(x); }      // Add to rear
  popFront()   { return this.items.shift(); }   // Remove from front
  popBack()    { return this.items.pop(); }    // Remove from rear
  peekFront()  { return this.items[0]; }
  peekBack()   { return this.items[this.items.length-1]; }
  isEmpty()    { return this.items.length === 0; }
  size()       { return this.items.length; }
}

// ─── Sliding Window Maximum — O(n) using a Deque ─────────────────
// Find the maximum value in every window of size k.
// The deque stores indices of "useful" candidates in decreasing order.
function maxSlidingWindow(nums, k) {
  const dq = new Deque();   // Stores indices (not values)
  const result = [];

  for (let i = 0; i < nums.length; i++) {

    // Remove indices that are out of this window
    while (!dq.isEmpty() && dq.peekFront() < i - k + 1)
      dq.popFront();

    // Remove smaller values from rear — they can never be the max
    while (!dq.isEmpty() && nums[dq.peekBack()] < nums[i])
      dq.popBack();

    dq.pushBack(i);

    // Window is fully formed; record the max (always at front)
    if (i >= k - 1) result.push(nums[dq.peekFront()]);
  }
  return result;
}

console.log(maxSlidingWindow([1,3,-1,-3,5,3,6,7], 3));
// [3, 3, 5, 5, 6, 7]

Plain English Analogy

🎮

Standard Queue — Waiting to Buy Tickets at a Gaming Tournament

You're at a gaming expo, and there's a huge line to get your copy signed by a pro gamer. The first person to join the line is the first one to get their signature and leave. Nobody can cut the queue or jump out from the middle. If you arrive last, you wait longest. That's FIFO — the rule that makes queues fair.

  • Joining the back of the line → enqueue()
  • The person at the front getting served and leaving → dequeue()
  • Checking who's next without moving them → peek()
🎡

Circular Queue — A Looping Rollercoaster Ride

Imagine a rollercoaster with exactly 6 seats arranged in a circle. Once a seat is vacated at the front, that same seat becomes available again at the back of the cycle — no new track is added. The ride keeps looping through the same fixed seats. A circular queue does the same thing with array slots: once you dequeue from the front, that slot wraps around and becomes usable at the rear again, using modular arithmetic instead of a new seat.

This is critical when you have a limited, fixed-size buffer and can't afford to waste any space.

🚌

Deque — A Bus with Doors at Both Ends

Most buses have a door at the front (to board) and a door at the back (to exit). But imagine a special VIP bus where people can board and exit from either end. A VIP can board from the front; everyone else boards from the back. A deque works exactly the same — you choose which end to use depending on whether you need stack behaviour (same end) or queue behaviour (opposite ends), and you can mix both in the same data structure.

  • VIP boarding at the front → pushFront()
  • Regular boarding at the back → pushBack()
  • Exiting from either door → popFront() / popBack()

Real-World Coding Applications

🖨️

Print Spooler

Documents sent to a printer are held in a queue. The first document sent is the first to print — FIFO ensures jobs aren't starved or reordered.

🌐

Web Server Request Queue

HTTP servers queue incoming requests when under load. Each worker thread dequeues the next request to handle. Fair, orderly, and essential for high traffic.

📡

Network Packet Buffers

Routers use circular queues (ring buffers) to hold incoming packets. When the buffer fills, new packets are dropped — a bounded, zero-wasted-space design.

🔍

BFS Graph Traversal

Breadth-First Search uses a queue to visit nodes level by level — closest nodes first. Used in shortest-path algorithms like those behind Google Maps.

🪟

Sliding Window Algorithms

Deques power O(n) sliding window problems: finding moving averages, maximum in a window, or rate-limiting a stream of events in real-time systems.

⌨️

Keyboard Input Buffer

Your keyboard's keystrokes are stored in a circular queue in hardware. Characters are consumed by the OS in the order they were typed — FIFO guaranteed.
