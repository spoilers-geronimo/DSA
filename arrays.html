Formal Definition

What is an Array?

An array is a linear data structure that stores a fixed-size, ordered collection of elements of the same data type in contiguous (adjacent) memory locations. Each element is identified by a non-negative integer index, starting at 0. Given a base address and element size, any element can be located in O(1) time using: address(A[i]) = base + i × size.

— Standard Computer Science Definition
2
Explanation

How Arrays Actually Work

When you declare an array, the OS reserves a single contiguous block of memory large enough for all elements. If each integer is 4 bytes and your array has 5 elements, exactly 20 consecutive bytes are allocated.

Because every element is the same size and they're packed side-by-side, the CPU can jump directly to any element using one arithmetic calculation — no traversal needed. That's why random access is O(1).

The tradeoff: inserting or deleting in the middle forces every subsequent element to shift by one position — that's up to O(n) work. And arrays are fixed in size; resizing means allocating a brand-new block and copying everything over.

Code Snippet

Arrays in JavaScript

A complete walkthrough of every core array operation — creation, access, update, insertion, deletion, and search.

JavaScript 
// 1. Create an array — 5 elements, indices 0..4
const scores = [85, 92, 78, 95, 88];
//               [0]  [1]  [2]  [3]  [4]

// 2. Access by index — O(1)
console.log(scores[0]);       // 85
console.log(scores[3]);       // 95
console.log(scores.length);   // 5

// 3. Update an element — O(1)
scores[2] = 82;
// scores → [85, 92, 82, 95, 88]

// 4. Append to end — O(1)
scores.push(91);
// scores → [85, 92, 82, 95, 88, 91]

// 5. Insert in the middle — O(n)  (elements must shift right)
scores.splice(2, 0, 99);
// scores → [85, 92, 99, 82, 95, 88, 91]

// 6. Remove last element — O(1)
scores.pop();
// scores → [85, 92, 99, 82, 95, 88]

// 7. Remove from the middle — O(n)  (elements must shift left)
scores.splice(2, 1);
// scores → [85, 92, 82, 95, 88]

// 8. Traverse — O(n)
for (let i = 0; i < scores.length; i++) {
  console.log(`index ${i} → ${scores[i]}`);
}

// 9. Linear search — O(n)
function linearSearch(arr, target) {
  for (let i = 0; i < arr.length; i++) {
    if (arr[i] === target) return i;  // found
  }
  return -1;  // not found
}

console.log(linearSearch(scores, 95)); // 3
console.log(linearSearch(scores, 10)); // -1

Analogy for a 14-year-old

Think of It Like a Numbered Pencil Case

🎒 The Pencil Case Analogy

Imagine a pencil case with exactly 5 numbered slots — slots 0, 1, 2, 3, and 4. Each slot holds exactly one pen.

Your friend asks: "What's in slot 3?" You don't open every slot — you go directly to slot 3 and grab it instantly. That's O(1) access — you know the exact location.

But what if you want to add a new pen between slot 1 and slot 2? You'd have to slide every pen from slot 2, 3, and 4 one position to the right to make room. The more pens, the more sliding — that's O(n) insertion.

And the biggest limitation: your pencil case has a fixed number of slots. If it's full and you need a 6th slot, you'd need to buy a bigger pencil case and transfer everything over. That's exactly what happens when an array runs out of space.

🎯 Key Takeaways

  • Elements are stored side-by-side in one block of memory (contiguous).
  • Any element can be accessed in O(1) using its index — no searching.
  • Inserting or deleting in the middle is O(n) — elements must shift.
  • Arrays have a fixed size; dynamic resizing copies everything to a new block.
  • Best used when you need fast random access and the size is known upfront.
