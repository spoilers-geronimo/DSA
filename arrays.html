Formal Definition

An Array is a linear data structure that stores a collection of elements of the same type in contiguous memory locations. Each element is identified by a numeric index, beginning at zero. A static array has a fixed size determined at allocation time; a dynamic array (such as JavaScript's Array or Python's list) can resize itself automatically as elements are added or removed.

The defining property of an array is random access: given any valid index, the element at that position can be read or written in constant time, because the memory address is computed directly as base_address + (index × element_size).

2

Formal Explanation & Operations

🔒 Static Array

  • Fixed size set at creation
  • Memory allocated once (stack or heap)
  • No overhead — compact, fast
  • Cannot grow or shrink
  • Common in C, C++, Java primitives

🔄 Dynamic Array

  • Grows automatically when full
  • Internally uses a static array
  • When full, allocates ~2× capacity and copies
  • Amortised O(1) append
  • JS Array, Python list, C++ vector
Operation Description Static Dynamic
arr[i] Read or write element at index i O(1) O(1)
Append (end) Add element to the last position O(n)* O(1) amort.
Insert (middle) Insert at index i, shift elements right O(n) O(n)
Delete (middle) Remove at index i, shift elements left O(n) O(n)
Search (unsorted) Linear scan to find a value O(n) O(n)
Search (sorted) Binary search on a sorted array O(log n) O(log n)

* Static arrays cannot append if full — a new larger array must be manually allocated.

💡

Why is access O(1)? The CPU computes the exact memory address of arr[i] as base + i × sizeof(element) in a single arithmetic instruction — no searching required, no matter how large the array is.

Code Snippet

The first example demonstrates static-like array usage and core operations. The second implements a minimal dynamic array from scratch to show how resizing works under the hood.

JavaScript — Core Array Operations 
// ─── Static-style Array (fixed-length) ──────────────────────────
const staticArr = new Array(5).fill(0);   // [0, 0, 0, 0, 0]
staticArr[0] = 10;
staticArr[2] = 30;
staticArr[4] = 50;
console.log(staticArr);              // [10, 0, 30, 0, 50]
console.log(staticArr[2]);            // 30  — O(1) random access

// ─── Dynamic Array operations ───────────────────────────────────
const nums = [10, 20, 30, 40];

// Append — O(1) amortised
nums.push(50);                         // [10, 20, 30, 40, 50]

// Insert at index 2 — O(n): elements after index shift right
nums.splice(2, 0, 99);                 // [10, 20, 99, 30, 40, 50]

// Delete at index 2 — O(n): elements after index shift left
nums.splice(2, 1);                    // [10, 20, 30, 40, 50]

// Linear search — O(n)
const idx = nums.indexOf(30);          // 2

// Iteration — O(n)
for (let i = 0; i < nums.length; i++) {
  console.log(`nums[${i}] = ${nums[i]}`);
}

// ─── Binary Search on a sorted array — O(log n) ─────────────────
function binarySearch(arr, target) {
  let left = 0, right = arr.length - 1;

  while (left <= right) {
    const mid = Math.floor((left + right) / 2);
    if      (arr[mid] === target) return mid;   // Found
    else if (arr[mid] < target)  left  = mid + 1; // Search right half
    else                         right = mid - 1; // Search left half
  }
  return -1; // Not found
}

const sorted = [2, 5, 8, 12, 16, 23, 38, 56];
console.log(binarySearch(sorted, 23));  // 5
console.log(binarySearch(sorted, 10));  // -1
JavaScript — Dynamic Array from Scratch 
// ─── DynamicArray: shows how resize/amortisation works ──────────
class DynamicArray {
  constructor() {
    this.data     = new Array(2);  // Start with capacity 2
    this.length   = 0;
    this.capacity = 2;
  }

  get(index) {
    if (index < 0 || index >= this.length)
      throw new RangeError("Index out of bounds");
    return this.data[index];
  }

  push(value) {
    if (this.length === this.capacity) {
      this._resize();   // Double capacity when full
    }
    this.data[this.length] = value;
    this.length++;
  }

  _resize() {
    this.capacity *= 2;                      // Double the capacity
    const newData = new Array(this.capacity);
    for (let i = 0; i < this.length; i++) {
      newData[i] = this.data[i];             // Copy all existing elements
    }
    this.data = newData;
    console.log(`Resized to capacity: ${this.capacity}`);
  }
}

const da = new DynamicArray();
da.push(1);  // capacity: 2
da.push(2);  // capacity: 2
da.push(3);  // → Resized to capacity: 4
da.push(4);  // capacity: 4
da.push(5);  // → Resized to capacity: 8
console.log(da.get(2)); // 3

Plain English Analogy

🎬

A Row of Numbered Seats in a Cinema

Imagine a row of cinema seats, each with a number printed on it: Seat 0, Seat 1, Seat 2, and so on. If someone tells you "your friend is in seat 4", you walk directly to seat 4 — you don't check every seat from the beginning. That's O(1) random access.

A static array is like a row with exactly 10 seats — the cinema was built that way and the walls can't be moved. A dynamic array is like a flexible venue: when every seat is full and a new person arrives, the whole group moves to a room with twice as many seats.

  • Seat number = index
  • Person in the seat = element
  • "Go directly to seat 4" = O(1) access
  • Moving everyone to a bigger room = dynamic resize (O(n), but rare)

Real-World Coding Applications

Arrays are so foundational that nearly every other data structure is built on top of them.

🖼️

Image Pixels

A digital image is a 2D array of pixels. Each cell stores RGB colour values. GPU shaders process these arrays in parallel at enormous speed.

📊

Spreadsheets & Databases

Table rows and columns in SQL databases and Excel are backed by arrays. Column-oriented databases store each column as a flat array for fast aggregation.

🔊

Audio & Video Buffers

Audio waveforms are arrays of samples (e.g. 44,100 per second). Video frames are arrays of pixel arrays. Media players stream these at precise intervals.

🧮

Machine Learning

Neural networks operate entirely on multi-dimensional arrays (tensors). Libraries like NumPy and PyTorch wrap highly optimised C array operations.

🗃️

Hash Tables

The internal backing structure of a hash map (JavaScript objects, Python dicts) is an array. The hash function maps keys to array indices.

🔀

Sorting Algorithms

Almost every sorting algorithm (quicksort, mergesort, heapsort) operates on arrays. The contiguous layout enables cache-efficient comparisons and swaps.
