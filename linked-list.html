Singly & Doubly O(1) Insert/Delete Pointer-Based Dynamic Size

Formal Definition

A Linked List is a linear data structure in which elements (called nodes) are stored in non-contiguous memory locations. Each node contains two fields: a data field holding the stored value, and a pointer (or reference) field pointing to the next node in the sequence. The list is accessed through a reference to its first node, called the head. The last node's pointer is null, signalling the end of the list. A doubly linked list augments each node with an additional pointer to its predecessor, enabling bidirectional traversal.

Unlike arrays, linked lists do not require contiguous memory. Each node can sit anywhere in the heap. The trade-off: random access is gone — to reach node k, you must follow pointers from the head, one step at a time.

Plain English Explanation & Operations

➡️ Singly Linked List

  • Each node holds data + one pointer (next)
  • Traversal in one direction only (head → tail)
  • Deleting a node requires the previous node's pointer
  • Less memory — one pointer per node
  • Used for stacks, simple queues, undo history

⇄ Doubly Linked List

  • Each node holds data + two pointers (prev, next)
  • Traversal in both directions
  • Delete any node in O(1) with a direct reference to it
  • More memory — two pointers per node
  • Used for browser history, LRU cache, deques
OperationDescriptionSinglyDoubly
Access by index Reach the k-th node O(n) O(n)
Prepend (head) Insert a new node at the front O(1) O(1)
Append (tail) Insert at the end (with tail pointer) O(1) O(1)
Insert (middle) Insert after a given node reference O(1)* O(1)*
Delete (head) Remove the first node O(1) O(1)
Delete (given reference) Remove a node you already have a pointer to O(n)** O(1)
Search Find first node with a given value O(n) O(n)

* You must have a reference to the preceding node. Finding that node is O(n).
** Singly lists need to locate the previous node first — that walk is O(n).

💡

Array vs Linked List — the core trade-off: Arrays give you O(1) access at any index, but insertions/deletions in the middle are O(n) due to shifting. Linked lists flip this: insert or delete anywhere in O(1) once you have the node, but accessing the k-th element costs O(n) because you must walk the chain.

⚠️

Cache locality: Because linked list nodes are scattered in heap memory, the CPU cache cannot prefetch them the way it can with array elements. In practice, arrays are often faster even for insert-heavy workloads because of this cache advantage.

Code Snippet

The first snippet builds a minimal singly linked list class with all core operations. The second shows a doubly linked list — the foundation of most real-world list implementations.

JavaScript — Singly Linked List 
// ─── Node blueprint ─────────────────────────────────────────────
class Node {
  constructor(value) {
    this.value = value;
    this.next  = null;   // pointer to the next node
  }
}

// ─── Singly Linked List ──────────────────────────────────────────
class LinkedList {
  constructor() {
    this.head   = null;
    this.tail   = null;
    this.length = 0;
  }

  // O(1) — insert at end
  append(value) {
    const node = new Node(value);
    if (!this.head) {
      this.head = this.tail = node;
    } else {
      this.tail.next = node;   // wire old tail → new node
      this.tail      = node;   // advance tail pointer
    }
    this.length++;
  }

  // O(1) — insert at front
  prepend(value) {
    const node = new Node(value);
    node.next  = this.head;  // new node points to old head
    this.head   = node;
    if (!this.tail) this.tail = node;
    this.length++;
  }

  // O(n) — delete by value
  delete(value) {
    if (!this.head) return;
    if (this.head.value === value) {
      this.head = this.head.next;  // skip head
      this.length--;
      return;
    }
    let curr = this.head;
    while (curr.next) {
      if (curr.next.value === value) {
        if (curr.next === this.tail) this.tail = curr;
        curr.next = curr.next.next;  // bypass the target node
        this.length--;
        return;
      }
      curr = curr.next;
    }
  }

  // O(n) — access by index
  get(index) {
    let curr = this.head, i = 0;
    while (curr) {
      if (i === index) return curr.value;
      curr = curr.next;
      i++;
    }
    return null;
  }

  // Utility: convert to array for printing
  toArray() {
    const result = [];
    let curr = this.head;
    while (curr) { result.push(curr.value); curr = curr.next; }
    return result;
  }
}

// ─── Usage ───────────────────────────────────────────────────────
const list = new LinkedList();
list.append(10);
list.append(20);
list.append(30);
list.prepend(5);
console.log(list.toArray());   // [5, 10, 20, 30]
list.delete(20);
console.log(list.toArray());   // [5, 10, 30]
console.log(list.get(1));       // 10
JavaScript — Doubly Linked List 
class DNode {
  constructor(value) {
    this.value = value;
    this.prev  = null;
    this.next  = null;
  }
}

class DoublyLinkedList {
  constructor() {
    this.head = null;
    this.tail = null;
  }

  append(value) {
    const node = new DNode(value);
    if (!this.tail) {
      this.head = this.tail = node;
    } else {
      node.prev      = this.tail;  // back-pointer to old tail
      this.tail.next = node;
      this.tail      = node;
    }
  }

  // O(1) delete — we already have the node, no walk needed
  deleteNode(node) {
    if (node.prev) node.prev.next = node.next;
    else            this.head       = node.next;

    if (node.next) node.next.prev = node.prev;
    else            this.tail       = node.prev;

    node.prev = node.next = null;  // help GC
  }
}

const dll = new DoublyLinkedList();
dll.append('A');
dll.append('B');
dll.append('C');
// dll.head → A ⇄ B ⇄ C ← dll.tail
dll.deleteNode(dll.head.next);   // delete B in O(1)
// dll.head → A ⇄ C ← dll.tail

Plain English Analogy

🚂

A Train with Detachable Carriages

Picture a freight train. Each carriage (node) carries some cargo (data) and is attached to the next carriage with a coupling hook (pointer). The train driver only knows where the first carriage is — the head.

Now compare this to a cinema row (arrays). In the cinema, every seat is in a fixed, numbered spot. You can jump straight to seat 7. On the train, to reach the 7th carriage, you have to walk through carriages 1, 2, 3… one at a time — that's O(n) access.

But here's where trains win: adding or removing a carriage in the middle takes one second. You unhook one coupling and re-hook two others. No seats need to shift. With a cinema row, every person after the gap has to move one seat — that's O(n) for a linked list insert.

  • Carriage = node
  • Cargo inside = the data value
  • Coupling hook = the next pointer
  • Engine (loco) = the head reference
  • Rear carriage = the tail reference
  • Unhook + rehook = O(1) insert / delete

Real-World Coding Applications

Linked lists appear wherever you need fast insertion or deletion and don't need random index access.

🌐

Browser History

Each page you visit is a node in a doubly linked list. The Back and Forward buttons simply walk prev/next pointers — O(1) navigation.

↩️

Undo / Redo

Text editors (VS Code, Photoshop) store edit operations in a doubly linked list. Undo pops from the tail; redo follows the forward pointer.

📦

Memory Allocators

The OS's free-list (blocks of available heap memory) is a linked list. When you call malloc(), the allocator walks this list and splices in your block.

⚡

LRU Cache

A doubly linked list + hash map powers every LRU (Least Recently Used) cache. Moving a recently-used item to the front is O(1) with a node reference.

🎵

Music / Video Playlist

A playlist is a doubly linked list. Jump to previous track (prev) or next track (next) in O(1). Shuffle by rewiring a few pointers.

🔗

Hash Table Chaining

When two keys hash to the same bucket, each bucket stores a singly linked list of all colliding entries. This is the "chaining" collision strategy used in Java's HashMap.

Classic Interview Question

🔁 Reverse a Singly Linked List

Given the head of a singly linked list, reverse the list in-place and return the new head. You must do it in O(n) time and O(1) extra space.

⚡ Easy — but asked in ~70% of linked list rounds

Why this question? It's simple enough to describe in one sentence but requires genuine understanding of pointer manipulation. A candidate who truly understands how next pointers work solves it cleanly; one who doesn't will tie themselves in knots.

The approach — three-pointer technique:

  • 1
    Keep three pointers: prev (starts null), curr (starts at head), next (temporary).
  • 2
    Save curr.next into next before you overwrite it.
  • 3
    Point curr.next backwards to prev — this is the reversal step.
  • 4
    Advance prev = curr and curr = next, then repeat.
  • 5
    When curr is null, prev is the new head.
JavaScript — Reverse Linked List 
// ─── Iterative — O(n) time, O(1) space ─────────────────────────
function reverseList(head) {
  let prev = null;
  let curr = head;

  while (curr !== null) {
    const next = curr.next;   // 1. save next before we overwrite it
    curr.next  = prev;          // 2. reverse the arrow
    prev        = curr;          // 3. advance prev
    curr        = next;          // 4. advance curr
  }

  return prev;  // prev is now the new head
}

// ─── Walkthrough: 1 → 2 → 3 → null ─────────────────────────────
// Start:   prev=null, curr=1
// Step 1:  next=2,  1.next=null,  prev=1, curr=2
// Step 2:  next=3,  2.next=1,     prev=2, curr=3
// Step 3:  next=null, 3.next=2,   prev=3, curr=null
// Done:    return 3  →  3 → 2 → 1 → null  ✓

// ─── Bonus: Recursive version (O(n) time, O(n) stack space) ─────
function reverseListRecursive(head) {
  if (!head || !head.next) return head;  // base case

  const newHead = reverseListRecursive(head.next);
  head.next.next = head;   // make next node point back
  head.next     = null;   // cut forward pointer
  return newHead;
}
✅

Interview tip: Walk through the algorithm on a 3-node example before coding. Draw the pointer changes on a whiteboard. Interviewers want to see you reason about pointer mutations — off-by-one bugs here are a red flag. Also mention the recursive version to show depth, but note its O(n) stack space cost.
