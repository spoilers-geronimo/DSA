Formal Definition

What is a Linked List?

A linked list is a linear data structure in which elements (called nodes) are stored at non-contiguous memory locations. Each node contains two fields: a data field holding the stored value, and a next pointer holding the memory address of the next node. The list begins at a pointer called the head. The final node's next pointer is null, marking the end of the list.

— Standard Computer Science Definition

Variants: a Singly Linked List (each node points only forward), a Doubly Linked List (each node points forward and backward), and a Circular Linked List (the tail points back to the head).

2
Explanation

How Linked Lists Work

Unlike arrays, linked lists don't need a contiguous block of memory. Each node can live anywhere — they're connected purely by pointers (memory addresses) saying "the next node is over there."

This makes insertion and deletion at the head O(1) — you just rewire a pointer, no shifting. But to reach the 5th element you must walk the chain from the head one node at a time, making random access O(n).

Code Snippet

Singly Linked List in JavaScript

Full implementation: Node class, prepend, append, delete, and print operations.

JavaScript 
// ── Node: the single building block ──────────────────
class Node {
  constructor(data) {
    this.data = data;  // the stored value
    this.next = null;  // pointer to next node (null = no next)
  }
}

// ── LinkedList: manages the chain ────────────────────
class LinkedList {
  constructor() {
    this.head = null;  // entry point of the list
    this.size = 0;
  }

  // Insert at the front — O(1)
  prepend(data) {
    const node = new Node(data);
    node.next = this.head;  // new node points to old head
    this.head = node;       // head now points to new node
    this.size++;
  }

  // Insert at the back — O(n)
  append(data) {
    const node = new Node(data);
    if (!this.head) {
      this.head = node;      // empty list: new node becomes head
    } else {
      let curr = this.head;
      while (curr.next) {
        curr = curr.next;    // walk to the last node
      }
      curr.next = node;      // attach new node at the end
    }
    this.size++;
  }

  // Delete first node with matching data — O(n)
  delete(data) {
    if (!this.head) return;

    // Special case: target is the head node
    if (this.head.data === data) {
      this.head = this.head.next;
      this.size--;
      return;
    }

    // Walk until we find the node before the target
    let curr = this.head;
    while (curr.next) {
      if (curr.next.data === data) {
        curr.next = curr.next.next;  // bypass the target node
        this.size--;
        return;
      }
      curr = curr.next;
    }
  }

  // Print list as: 5 → 10 → 20 → null
  print() {
    const parts = [];
    let curr = this.head;
    while (curr) {
      parts.push(curr.data);
      curr = curr.next;
    }
    console.log(parts.join(' → ') + ' → null');
  }
}

// ── Demo ─────────────────────────────────────────────
const list = new LinkedList();
list.append(10);
list.append(20);
list.append(30);
list.prepend(5);
list.print();    // 5 → 10 → 20 → 30 → null

list.delete(10);
list.print();    // 5 → 20 → 30 → null

console.log(list.size); // 3

Analogy for a 14-year-old

Think of It Like a Scavenger Hunt

🗺️ The Scavenger Hunt Analogy

Your teacher sets up a scavenger hunt around school. You get a clue card at the entrance that says: "Your next clue is in the library."

You go to the library. That clue says: "Your next clue is in the gym." The gym clue says "Room 12." And so on.

That's a linked list! Each location (node) holds a piece of data (the clue) and a pointer (where to go next). The clues are scattered all over school — not lined up in order like lockers.

If the teacher wants to insert a new clue between the library and the gym, she just changes the library clue to point to "Cafeteria" and the cafeteria clue to point to "Gym." No reprinting every clue — just update two cards. That's O(1) insertion!

But if someone asks "what's clue #4?" you have to follow every clue from the start until you count to 4. You can't teleport. That's why access is O(n).

🎯 Key Takeaways

  • Each node stores data AND a pointer (next) to the following node.
  • Nodes can live anywhere in memory — no contiguous block needed.
  • Inserting at the head is O(1) — just update the head pointer.
  • Random access requires walking from the head — O(n).
  • Each node uses extra memory for the pointer (8 bytes on 64-bit systems).
  • Perfect when you need frequent head insertions/deletions and don't need random access.
