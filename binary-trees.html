Formal Definition

A Binary Tree is a hierarchical data structure in which each node contains a value and at most two children, referred to as the left child and the right child. The topmost node is the root; nodes with no children are leaves. The height of a tree is the length of the longest path from root to a leaf. A Binary Search Tree (BST) adds an ordering invariant: for every node, all values in its left subtree are strictly less than the node's value, and all values in its right subtree are strictly greater, enabling O(log n) search in a balanced tree.

The key distinction between trees and lists is non-linearity: each node can branch into two paths. This branching is what gives balanced binary trees their O(log n) operations — at every step, half the remaining nodes are eliminated from consideration.

2

Plain English Explanation & Operations

Every node in a binary tree is itself the root of a smaller tree — its subtree. This self-similar structure makes trees naturally recursive: almost every tree algorithm is a function that calls itself on the left and right subtrees.

🌳 Binary Tree (General)

  • Each node has at most 2 children
  • No ordering constraint on values
  • Used for expression trees, syntax trees
  • Traversal: in-order, pre-order, post-order

🔍 Binary Search Tree (BST)

  • Left child < parent < right child
  • In-order traversal yields sorted sequence
  • Search, insert, delete: O(log n) if balanced
  • Degrades to O(n) if unbalanced (skewed)

⚖️ Balanced BST (AVL / Red-Black)

  • Self-balancing after every insert/delete
  • Guarantees height ≈ log n at all times
  • All operations stay O(log n) worst-case
  • Used by Java's TreeMap, C++ std::map

✅ Complete / Perfect Tree

  • Complete: all levels filled except possibly the last, filled left to right
  • Perfect: all internal nodes have 2 children, all leaves at same level
  • n nodes in a perfect tree → height = log₂(n+1) − 1
  • Heap property relies on complete tree shape
OperationDescriptionBalanced BSTUnbalanced BST
Search Find a value in the tree O(log n) O(n)
Insert Add a new value maintaining BST order O(log n) O(n)
Delete Remove a node and restructure O(log n) O(n)
Traversal Visit every node once (any order) O(n) O(n)
Height / Depth Longest root-to-leaf path O(log n) O(n)
💡

The three traversals: In-order (left → root → right) visits BST nodes in sorted ascending order. Pre-order (root → left → right) is used to copy or serialize a tree. Post-order (left → right → root) is used to delete a tree or evaluate expression trees — the root is always processed last.

⚠️

Beware the degenerate case: If you insert values into a BST in sorted order (1, 2, 3, 4, 5…), every node has only a right child — the tree becomes a linked list with O(n) operations. Self-balancing trees (AVL, Red-Black) exist precisely to prevent this.

Code Snippet

The first snippet builds a BST with insert and search. The second implements all three classic traversals — each one a natural recursive function.

JavaScript — Binary Search Tree 
class TreeNode {
  constructor(val) {
    this.val   = val;
    this.left  = null;
    this.right = null;
  }
}

class BST {
  constructor() { this.root = null; }

  // O(log n) average — insert while maintaining BST order
  insert(val) {
    const node = new TreeNode(val);
    if (!this.root) { this.root = node; return; }

    let curr = this.root;
    while (true) {
      if (val < curr.val) {
        if (!curr.left)  { curr.left  = node; return; }
        curr = curr.left;
      } else {
        if (!curr.right) { curr.right = node; return; }
        curr = curr.right;
      }
    }
  }

  // O(log n) average — search by value
  search(val) {
    let curr = this.root;
    while (curr) {
      if      (val === curr.val) return true;
      else if (val <  curr.val) curr = curr.left;
      else                       curr = curr.right;
    }
    return false;
  }
}

// ─── Build a BST ─────────────────────────────────────────────────
const bst = new BST();
[8, 4, 12, 2, 6, 10, 14].forEach(v => bst.insert(v));

//        8          ← root
//       / \
//      4   12
//     / \ /  \
//    2  6 10  14   ← leaves

console.log(bst.search(6));   // true
console.log(bst.search(7));   // false
JavaScript — Three Traversals (Recursive) 
// ─── In-order: LEFT → ROOT → RIGHT ──────────────────────────────
// On a BST, this yields values in sorted (ascending) order.
function inOrder(node, result = []) {
  if (!node) return result;
  inOrder(node.left,  result);   // recurse left
  result.push(node.val);          // visit root
  inOrder(node.right, result);   // recurse right
  return result;
}

// ─── Pre-order: ROOT → LEFT → RIGHT ─────────────────────────────
// Useful for cloning a tree or serialising its structure.
function preOrder(node, result = []) {
  if (!node) return result;
  result.push(node.val);          // visit root FIRST
  preOrder(node.left,  result);
  preOrder(node.right, result);
  return result;
}

// ─── Post-order: LEFT → RIGHT → ROOT ────────────────────────────
// Root visited LAST — ideal for deletion or evaluating expressions.
function postOrder(node, result = []) {
  if (!node) return result;
  postOrder(node.left,  result);
  postOrder(node.right, result);
  result.push(node.val);          // visit root LAST
  return result;
}

// ─── Results on the BST above ────────────────────────────────────
console.log(inOrder(bst.root));    // [2, 4, 6, 8, 10, 12, 14] ← sorted!
console.log(preOrder(bst.root));   // [8, 4, 2, 6, 12, 10, 14]
console.log(postOrder(bst.root));  // [2, 6, 4, 10, 14, 12, 8]

Plain English Analogy

📖

A "Higher or Lower" Number Guessing Game

You know the game: one person thinks of a number between 1 and 100, and you guess. They say "higher" or "lower" each time. A great strategy is to always guess the middle — this is binary search, and a Binary Search Tree is the physical data structure that encodes this strategy.

Imagine the game frozen as a tree. The first guess (50) is the root. If the answer is lower, you branch left to 25; if higher, you branch right to 75. Each branch leads to a new subtree that covers half the remaining space. Within just 7 guesses you can find any number between 1 and 100 — because log₂(100) ≈ 7.

The three traversals also have intuitive meanings. Imagine exploring a family tree document:

  • Pre-order = read each family member before their descendants — great for printing the family structure
  • In-order = read members in birth order — great for listing them in a sorted way
  • Post-order = read descendants before their parent — great for calculating totals (sum up children first, then the parent)

Real-World Coding Applications

Binary trees and their variants appear at almost every level of a computing system.

🗂️

File Systems

Directory structures are trees. Your /home/user/docs/report.pdf path traces a root-to-leaf path. File explorers traverse this tree to render the hierarchy.

🔡

Compilers & Parsers

Source code is parsed into an Abstract Syntax Tree (AST). The expression 3 * (4 + 5) becomes a tree — post-order traversal evaluates it correctly.

🗄️

Database B-Trees

B-Trees (generalised binary trees with many children per node) are the primary index structure in every major database. MySQL's InnoDB, PostgreSQL — all use B+ Trees for range queries and sorted access.

🔠

Autocomplete & Tries

Autocomplete in search bars uses a Trie — a tree where each node is a character. Following a path from root to a node spells out a prefix, enabling O(k) prefix lookups where k = prefix length.

🤖

Decision Trees (ML)

Machine learning decision tree classifiers are literally binary trees. Each node asks a yes/no question about a feature; leaves are classification labels. Random Forests are ensembles of many such trees.

🗜️

Huffman Encoding

Data compression algorithms (ZIP, JPEG, MP3) build a binary tree to assign shorter bit codes to frequent characters. The tree's structure determines the compression ratio.

Classic Interview Question

📏 Maximum Depth of a Binary Tree

Given the root of a binary tree, return its maximum depth — the number of nodes along the longest path from the root down to the farthest leaf.

Example: the tree [3, 9, 20, null, null, 15, 7] has a maximum depth of 3.

⚡ Easy — LeetCode #104 · the canonical intro to recursive tree thinking

Why this question? It's the perfect entry point for recursive tree thinking — the pattern that unlocks hundreds of harder problems. Once you see that "depth of tree = 1 + max(depth of left, depth of right)", you've internalised the recursive decomposition that applies to diameter, path sum, lowest common ancestor, and beyond.

The approach — recursive decomposition:

  • 1
    Base case: if the node is null, its depth is 0.
  • 2
    Recursively compute the depth of the left subtree.
  • 3
    Recursively compute the depth of the right subtree.
  • 4
    Return 1 + Math.max(leftDepth, rightDepth) — the current node adds 1 to the deeper branch.
JavaScript — Maximum Depth of Binary Tree 
// ─── Recursive DFS — O(n) time, O(h) space (h = tree height) ────
function maxDepth(root) {
  if (!root) return 0;                       // base case: empty node

  const leftDepth  = maxDepth(root.left);    // depth of left subtree
  const rightDepth = maxDepth(root.right);   // depth of right subtree

  return 1 + Math.max(leftDepth, rightDepth); // current node adds 1
}

// ─── Walkthrough on tree [3, 9, 20, null, null, 15, 7] ──────────
//        3           maxDepth(3)
//       / \          = 1 + max( maxDepth(9), maxDepth(20) )
//      9  20         = 1 + max( 1,           2            )
//        /  \        = 1 + 2
//       15   7       = 3  ✓

// ─── Iterative BFS (level-order) — same complexity ───────────────
function maxDepthBFS(root) {
  if (!root) return 0;
  let depth = 0;
  const queue = [root];

  while (queue.length) {
    const levelSize = queue.length;     // all nodes at current level

    for (let i = 0; i < levelSize; i++) {
      const node = queue.shift();
      if (node.left)  queue.push(node.left);
      if (node.right) queue.push(node.right);
    }
    depth++;                              // finished one level
  }

  return depth;
}
✅

Interview tip: The recursive solution is three lines and deeply elegant — interviewers love it. Mention the space complexity: O(h) for the call stack, where h is the tree height (O(log n) for balanced, O(n) worst case for a skewed tree). Offering the iterative BFS alternative shows you know multiple approaches. The recursive pattern here — "solve for children, combine results, return to parent" — is the template for at least 50 LeetCode tree problems.
